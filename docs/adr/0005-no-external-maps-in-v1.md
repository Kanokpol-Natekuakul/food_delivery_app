# V1 ไม่พึ่ง Maps/Routing API ภายนอก

V1 จัดการเรื่องตำแหน่งและระยะทางด้วยตัวเองทั้งหมด ไม่เรียกใช้ Maps/Routing API ภายนอก:
- ที่อยู่ลูกค้าเก็บเป็นพิกัด lat/lng จากการ **ปักหมุดบนแผนที่** (ไม่ทำ geocoding จากที่อยู่ข้อความ)
- ระยะทางคิดแบบ **เส้นตรง (haversine)** ระหว่างพิกัดร้านกับลูกค้า (ไม่ใช่ระยะตามถนนจริง)
- ตำแหน่งไรเดอร์ใช้พิกัด query-time snapshot ตอนเปิดดูงาน (ดู ADR 0001 / Visibility Radius)

## เหตุผล

สอดคล้องกับแนวทาง "เบาไว้ก่อน" ของ V1 ทั้งระบบ — เลี่ยงค่าใช้จ่ายต่อครั้งและ external dependency ของ Maps API ในขณะที่ความแม่นยำระดับเส้นตรงเพียงพอสำหรับการคิดค่าส่งและกรองระยะในเฟสแรก

## Consequences

- **ระยะเส้นตรงต่ำกว่าระยะถนนจริงเสมอ** → ค่าส่งที่คิดได้จะต่ำกว่าระยะที่ไรเดอร์ขับจริง อาจต้องใส่ตัวคูณชดเชย (fudge factor) ในสูตรค่าส่ง
- การปักหมุดผิด (ลูกค้าปักพลาด) จะทำให้ทั้งค่าส่งและการกรอง Service Zone ผิด — ไม่มี geocoding มาตรวจทาน
- เป็นการตัดสินใจที่ตั้งใจเบี่ยงจากสิ่งที่ผู้อ่านคาดหวัง (แอปส่งอาหารมักใช้ระยะถนนจริง) — เมื่อจะอัปเกรดเป็น routing จริงต้องแก้ทั้งสูตรค่าส่งและการกรองระยะ
