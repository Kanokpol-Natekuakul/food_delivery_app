# ร้องเรียนหลังส่ง: goodwill เป็นค่าตั้งต้น + กู้การระบุความผิดด้วยสถิติ

ปัญหาที่ลูกค้าพบ *หลัง* รับของ (ผิดรายการ/อาหารเสียหาย/มีสิ่งแปลกปลอม) จัดการด้วย flow ร้องเรียนที่อยู่ **นอกวงจรชีวิตออเดอร์** (เกิดหลังออเดอร์ Completed): ลูกค้าร้องเรียนภายใน Complaint Window พร้อมแนบรูป เมื่อพิสูจน์ความผิดรายครั้งไม่ได้ แพลตฟอร์มคืนเงินแบบ goodwill จากกระเป๋าตัวเอง และติดตามสถิติร้องเรียนรายฝ่ายเพื่อจัดการพฤติกรรมในระยะยาว

## เหตุผล

เคสนี้ต่างจากการจบออเดอร์อื่นตรงที่ **มักพิสูจน์ความผิดรายครั้งไม่ได้** (อาหารถูกกิน/ทิ้งแล้ว ทั้งร้านและไรเดอร์ปฏิเสธ) หลัก "ใครผิดคนนั้นจ่าย" (ADR 0002) จึงใช้ตรงๆ ไม่ได้ ทางออกคือ:

- **รายครั้ง:** ถือว่า "ไม่มีใครผิดชัด" → แพลตฟอร์มแบก (goodwill) — เป็นการขยายหลักของ ADR 0002 ไม่ใช่ขัด
- **ระยะยาว:** ลายเซ็นทางสถิติกู้การระบุความผิดกลับมา — ร้าน/ไรเดอร์ที่โดนร้องซ้ำถูกลดอันดับ/ดำเนินการ และลูกค้าที่ร้องบ่อยผิดปกติถูกจับตา (กันโกง) แม้ไม่มีหลักฐานรายครั้ง

## Consequences

- ต้องมีระบบเก็บหลักฐาน (รูป) และ flow ร้องเรียนแยกจาก state machine ของออเดอร์
- ต้องเก็บอัตราร้องเรียนรายลูกค้า/ร้าน/ไรเดอร์ และนิยามเกณฑ์ "ผิดปกติทางสถิติ" (ยังไม่กำหนดตัวเลข)
- แพลตฟอร์มรับความเสี่ยงการโกงฝั่งลูกค้า (อ้างมั่วเพื่อขอ goodwill) — บรรเทาด้วยสถิติ ไม่ใช่การพิสูจน์รายครั้ง
- ความยาว Complaint Window ยังไม่กำหนด
