# โมเดลรายได้แบบผสม 3 ทาง

แพลตฟอร์มทำรายได้จาก 3 แหล่งพร้อมกันต่อหนึ่งออเดอร์: (1) ค่าคอมมิชชันหักจากค่าอาหารของร้าน, (2) ค่าบริการ (service fee) เก็บเพิ่มจากลูกค้า, และ (3) ส่วนแบ่งจากค่าส่ง โดยค่าส่งแพลตฟอร์มเป็นผู้กำหนดจากระยะทาง

ยอดที่ลูกค้าจ่าย = ค่าอาหาร + ค่าส่ง + ค่าบริการ
- ร้านได้รับ = ค่าอาหาร − ค่าคอมมิชชัน
- ไรเดอร์ได้รับ = ค่าส่ง − ส่วนแบ่งแพลตฟอร์ม
- แพลตฟอร์มได้รับ = ค่าคอมมิชชัน + ค่าบริการ + ส่วนแบ่งค่าส่ง

## เหตุผล

เลือกผสมหลายทางแทนแหล่งเดียว เพื่อกระจายการเก็บรายได้และไม่ให้ภาระตกหนักที่ฝ่ายใดฝ่ายหนึ่งจนเสียความสามารถในการแข่งขัน (เช่น ถ้าหักร้านหนักทางเดียว ร้านจะขึ้นราคาอาหารจนลูกค้าหนี)

## Consequences

- สูตรการแบ่งเงินมีหลายตัวแปร — ต้องคำนวณและบันทึกการแบ่งเงิน (money split) ต่อทุกออเดอร์ให้ตรวจสอบย้อนได้
- ยังไม่ได้นิยาม: อัตราคอมมิชชัน, สูตรค่าส่งตามระยะทาง, อัตราส่วนแบ่งค่าส่ง — เป็นพารามิเตอร์ที่ต้องกำหนดทีหลัง
- "คืนเต็ม" = คืนครบทั้ง 3 ก้อน (ค่าอาหาร + ค่าส่ง + ค่าบริการ) แพลตฟอร์มยอมไม่ได้ค่าบริการในเคสที่ออเดอร์ล้มเหลว ถือเป็นต้นทุนที่ยอมรับได้
