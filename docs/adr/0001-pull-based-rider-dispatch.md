# จัดส่งงานแบบ Pull (ไรเดอร์หยิบงานเอง) แทน Auto-dispatch

สำหรับ V1 งานจัดส่งจะถูกแสดงเป็นรายการให้ไรเดอร์ที่ว่างเห็นและกด "รับงาน" เอง (ใครรับก่อนได้ก่อน) แทนที่จะให้ระบบเลือกไรเดอร์ที่ใกล้ที่สุดและยิงงานให้อัตโนมัติ

เหตุผล: auto-dispatch ต้องมี tracking ตำแหน่งเรียลไทม์ อัลกอริทึมจับคู่ และการจัดการการปฏิเสธงาน ซึ่งหนักเกินไปสำหรับ V1 ส่วน pull model เริ่มต้นได้เบากว่ามาก

## Consequences

- ต้องออกแบบรับมือกรณี **ไม่มีไรเดอร์รับงาน** (ออเดอร์ค้างเติ่ง) เพราะไม่มีระบบบังคับมอบหมาย
- มี race condition: ไรเดอร์สองคนกดรับงานเดียวกันพร้อมกัน ต้องมีการล็อกเพื่อให้มีผู้ชนะคนเดียว
- ประสิทธิภาพการจับคู่ (ใกล้/ไกล) ไม่ได้ถูกปรับให้ดีที่สุด — แลกกับความเรียบง่าย
