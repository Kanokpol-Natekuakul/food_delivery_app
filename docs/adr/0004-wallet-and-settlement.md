# แพลตฟอร์มถือเงินกลาง + Wallet ภายใน + Settlement เป็นรอบ

ลูกค้าจ่ายเงินเข้าแพลตฟอร์มทั้งก้อนก่อน รายได้ส่วนของร้าน (ค่าอาหาร − คอม) และของไรเดอร์ (ค่าส่ง − ส่วนแบ่ง) จะถูกเครดิตเข้า **wallet ภายในระบบ** เมื่อออเดอร์สำเร็จ แล้วจึงโอนออกสู่บัญชีธนาคารจริงเป็นรอบ (settlement) ไม่ใช่จ่ายทันทีต่อทุกออเดอร์

## เหตุผล

- การโอนเงินจริงต่อทุกออเดอร์มีค่าธรรมเนียมสูงและจัดการ refund ยาก — wallet ภายในทำให้การคืนเงิน/หักลบเป็นเพียงรายการในบัญชี (ledger) ไม่ต้องโอนเงินจริงกลับไปกลับมา
- การเครดิตเข้า wallet "เมื่อออเดอร์สำเร็จ" ให้ผล escrow โดยอัตโนมัติ — เงินยังพักที่แพลตฟอร์มจนกว่าจะส่งของเสร็จ จึงรองรับ refund เต็มจำนวนตามโมเดลความรับผิด (ADR 0002) ได้ง่าย

## Consequences

- ต้องมีระบบบัญชี/ledger ภายในที่ตรวจสอบย้อนได้ (รายการเครดิต/เดบิต/refund ต่อ wallet)
- เงินของผู้ใช้ค้างอยู่ที่แพลตฟอร์มก่อน settlement — มีนัยด้านความน่าเชื่อถือและการกำกับดูแล (e-money/escrow) ที่ต้องพิจารณานอกขอบเขต V1
- ยังไม่ได้นิยาม: รอบ settlement (รายวัน/สัปดาห์/กดถอน), ยอดถอนขั้นต่ำ
