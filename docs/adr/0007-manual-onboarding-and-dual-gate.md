# การเข้าระบบ: ตรวจด้วยมือ + แยก work gate กับ payout gate

ร้านและไรเดอร์ต้องผ่าน **การตรวจเอกสารด้วยมือโดย Admin** ก่อนใช้งาน (V1 ไม่ต่อ KYC vendor ภายนอก) บัญชีมีสถานะ รอตรวจ → อนุมัติ → ระงับ และจะรับออเดอร์/คว้างานได้เฉพาะตอนสถานะ Approved การ "อนุมัติให้ทำงาน" (work gate) แยกจากการ "ถอนเงินได้" (payout gate): เริ่มทำงานและสะสมเงินใน wallet ได้ก่อน แต่ต้องยืนยันบัญชีธนาคารก่อนถอน/รับ settlement ครั้งแรก

## เหตุผล

- เป็น marketplace ที่จับทั้งอาหารและเงิน ปล่อยให้สมัครใช้ได้ทันที (instant self-serve) เสี่ยงร้านผี/ไรเดอร์ผี — จึงต้องมีการตรวจ
- เลือกตรวจด้วยมือแทน automated KYC vendor เพื่อให้สอดคล้องกับแนวทาง "V1 ไม่พึ่ง external service" (ADR 0005) — รับคอขวดด้านคนตรวจในช่วง volume ต่ำของ V1 ได้
- แยก work gate กับ payout gate เพื่อ **ลดแรงเสียดทานตอนสมัคร** — ไรเดอร์เริ่มหารายได้เข้า wallet ได้เลย (ADR 0004 ให้ผล escrow อยู่แล้ว) ค่อยผูกบัญชีตอนจะถอน

## Consequences

- การตรวจด้วยมือเป็นคอขวดเมื่อ scale — ต้องวางแผนเปลี่ยนเป็น automated เมื่อโต (จะขัด ADR 0005 ตอนนั้น ถือเป็น trade-off ที่ตั้งใจเลื่อน)
- ต้องมีหน้า Admin สำหรับ review + เก็บเอกสารที่อัปโหลด
- เงินค้างใน wallet ของผู้ที่ยังไม่ผูกบัญชี — ต้องมีนโยบายจัดการยอดค้างนาน (ยังไม่กำหนด)
