# โมเดลความรับผิดเมื่อออเดอร์ล้มเหลว: "ใครผิดคนนั้นจ่าย / ไม่มีใครผิดแพลตฟอร์มแบก"

เมื่อออเดอร์จบลงก่อนสำเร็จ ความเสียหาย (ค่าอาหารที่ทำทิ้ง, ค่าชดเชยไรเดอร์) จะถูกแบกตามหลักว่า "ฝ่ายที่ทำผิดเป็นผู้รับผิดชอบ ถ้าไม่มีฝ่ายใดผิด แพลตฟอร์มเป็นผู้แบก" ลูกค้าได้คืนเงินเต็มจำนวนเสมอในทุกกรณีที่ออเดอร์ไม่สำเร็จ

## เหตุผล

ต้องมีหลักการเดียวที่ใช้ตัดสินได้ทุกเคส แทนที่จะตัดสินเป็นรายกรณี เพื่อให้ behavior คาดเดาได้และเป็นธรรม โดยให้แพลตฟอร์มเป็น risk-bearer ของความเสี่ยงเชิงระบบ (เช่น ไม่มีไรเดอร์ในชั่วโมงเร่งด่วน) ที่ไม่ใช่ความผิดของลูกค้าหรือร้าน

## การประยุกต์กับแต่ละเคส

- **ร้านปฏิเสธ (Rejection):** ร้านผิด → ร้านจ่ายค่าชดเชยไรเดอร์, ลูกค้าคืนเต็ม, ร้านโดนลดอันดับ
- **ส่งไม่ได้ (Failed Delivery):** ลูกค้าผิด (ไม่อยู่รับ) → ลูกค้าไม่คืนเงิน, ไรเดอร์ได้ค่าส่ง, ร้านได้ค่าอาหาร, อาหารเป็นของไรเดอร์
- **ลูกค้ายกเลิกในหน้าต่างฟรี:** ไม่ถือว่าใครผิด → แพลตฟอร์มกลืนค่าอาหาร, ลูกค้าคืนเต็ม
- **Delivery Timeout (ไม่มีไรเดอร์):** ไม่มีใครผิด → แพลตฟอร์มกลืนค่าอาหาร, ลูกค้าคืนเต็ม
- **ไรเดอร์คว้าแล้วทิ้ง/หายบ่อย:** ไรเดอร์ผิด → ลดอันดับ + พักงาน (Rider Suspension)

## Consequences

- แพลตฟอร์มมีความเสี่ยงทางการเงินโดยตรงจากเคส "ไม่มีไรเดอร์" จึงต้องมีเครื่องมือคุมความเสี่ยง — V1 ใช้การจำกัดเวลา Delivery Timeout (Y) ให้สั้น
- ยังไม่ได้นิยาม: เกณฑ์ตัดสิน "ความผิด" ในเคสที่กำกวมกว่านี้ (เช่น ไรเดอร์รับงานแล้วหายไป)
