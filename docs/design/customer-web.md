# ดีไซน์ฝั่งลูกค้า — ทิศทาง "ตลาดกลางคืน" (Night Market)

เอกสารทิศทาง UX/UI สำหรับ **เว็ปฝั่งลูกค้า** (mobile-first) ของ marketplace ส่งอาหาร
อ้างอิงโดเมนที่ [CONTEXT.md](../../CONTEXT.md) และวงจรชีวิตออเดอร์ที่ [order-lifecycle.md](../order-lifecycle.md)

- **Subject:** marketplace ส่งอาหารตลาดไทย — โลกของสตรีทฟู้ด ตลาดกลางคืน วินมอเตอร์ไซค์ ป้ายไฟร้าน
- **Audience:** คนหิว เปิดมือถือตอนเย็น/ดึก อยากได้ของร้อนเร็วๆ
- **งานเดียวของหน้าแรก:** ทำให้ "เลือกร้านใกล้ฉันแล้วสั่งให้จบ" เกิดได้เร็วและน่ากิน

---

## 1. Thesis: ทำไมต้อง "ตลาดกลางคืน"

ความรู้สึกที่เป็นแก่นของ subject นี้คือ **เดินตลาดนัดตอนค่ำ** — ฉากมืด แต่แผงไฟส่องอาหารให้เรืองขึ้นมา ไอร้อนลอย มอเตอร์ไซค์มุด ของร้อนกำลังจะมาถึง ดีไซน์ทั้งหน้าจึงเป็น "ค่ำคืน" ที่อาหารและร้านเป็น **แสง** ลอยอยู่บนความมืด

> อาหารคือพระเอก ความมืดคือเวที — ยิ่งมืด อาหารยิ่งเรือง

## 2. ความเสี่ยงที่จงใจรับ (ต้อง justify)

**ธีมมืดสำหรับแอปอาหาร = เสี่ยง** — แอปอาหารส่วนใหญ่พื้นขาวสว่างเพราะเชื่อว่ากระตุ้นความอยาก ส่วนพื้นมืดถูกมองว่ากดความอยาก

**เหตุผลที่ยังเอา:** เพราะ subject เราเจาะจง "ตลาด**กลางคืน**" — ความมืดไม่ใช่อารมณ์เท่ๆ แต่คือ *ฉากจริง* ของการกินสตรีทฟู้ดตอนค่ำ และรูปอาหารอุ่นๆ ใต้แสงนีออนจะ "เด้ง" กว่าบนพื้นขาวเสมอ (เหมือนแผงร้านจริงที่เปิดไฟส่องกับข้าว) เรารับความเสี่ยงนี้ด้วยกติกาเดียว: **พื้นต้องเป็นดำ "อุ่น" (อมม่วง/อมน้ำตาล) ห้ามดำเทาเย็น** และรูปอาหารต้องอิ่มสีจัด

## 3. เช็คกับ default (กันหล่นเป็นของโหล)

AI ชอบออกแอปมืดเป็น "เกือบดำสนิท + สีสะท้อนแสงสีเดียว (เขียวมะนาว/แดงส้ม)" — เราเลี่ยง 3 จุด:

| default ที่เลี่ยง | สิ่งที่เราทำแทน |
|---|---|
| ดำเทากลาง #0A0A0A | ดำ **อมม่วงอุ่น** `#16101C` |
| accent สีเดียวเย็นๆ | นีออน **หลายสีแบบป้ายร้าน** (มะม่วง/พริก/ใบเตย) |
| accent ใช้เป็นเส้นขอบเฉยๆ | accent ใช้เป็น **แสงเรือง (glow)** ส่องการ์ด/ปุ่ม |

## 4. สี (Color tokens)

```
--ink        #16101C   ดำอมม่วงอุ่น — พื้นค่ำคืน (background)
--ink-card   #221829   พื้นการ์ด "แผงร้าน" ยกขึ้นจากพื้น
--mango      #FFB627   แสงป้ายไฟ — ไฮไลต์/display/glow หลัก
--chili      #FF4D3D   ความร้อน/ของด่วน — ปุ่มหลัก (สั่ง/เพิ่มลงตะกร้า)
--pandan     #57B368   สด/พร้อม/สำเร็จ — สถานะ available, ส่งสำเร็จ
--paper      #F4ECE0   ตัวอักษรหลักบนพื้นมืด (ขาวอุ่น ไม่ใช่ขาวจัด)
--paper-dim  #9C8FA6   ตัวอักษรรอง/คำอธิบาย
```

**glow tokens (signature):**
```
--glow-mango  0 0 28px rgba(255,182,39,.40)
--glow-chili  0 0 24px rgba(255,77,61,.45)
```

**คู่สีที่ผ่าน contrast (ใช้ตามนี้):**
- ข้อความ: `--paper` บน `--ink`/`--ink-card` (อัตราส่วนสูง ผ่านสบาย)
- ปุ่มหลัก: พื้น `--chili` + อักษร `--ink` (เข้มบนแดงสด) ขนาด ≥16px ตัวหนา
- ตัวเลข/ราคาเด่น: `--mango` บน `--ink` (เด้ง อ่านง่าย)
- ห้าม: `--paper-dim` เป็นข้อความสำคัญ (ใช้ได้เฉพาะ caption)

## 5. ตัวอักษร (Typography)

| บทบาท | ฟอนต์ | ใช้ที่ไหน |
|---|---|---|
| **Display** | **Kanit** 800 | ป้ายไฟ/หัวเรื่องใหญ่ — ใช้แบบประหยัด เน้นจุดเดียว |
| **UI/Heading** | **Anuphan** 600 | ชื่อร้าน ชื่อเมนู หัวข้อ section |
| **Body** | **Anuphan** 400 | คำอธิบาย เนื้อหา |
| **Data/Mono** | **Space Mono** | ราคา เลขออเดอร์ OTP ระยะทาง ตัวจับเวลา |

**ทำไมคู่นี้:** Kanit ให้พลังแบบป้ายร้านไทยตัวหนาทรงสูง · Anuphan เป็น grotesk ไทยที่มีคาแรกเตอร์แต่เคลียร์ (ไม่กลางจนจืด) · Space Mono ทำให้ "ราคา/ออเดอร์" รู้สึกเป็น **ใบเสร็จ/ข้อมูลจริง** แม้อยู่บนธีมมืด

**type scale (mobile):**
```
Display XL   clamp(2.4rem, 9vw, 4rem)   Kanit 800   leading .95  tracking -.02em
Display L    1.75rem                     Kanit 700   leading 1.05
Heading      1.125rem                    Anuphan 600
Body         1rem (16–17px)              Anuphan 400 leading 1.6
Label        .8125rem                    Anuphan 500  สีต่าง/หนัก (ไม่พึ่งตัวพิมพ์ใหญ่)
Data         .9375rem                    Space Mono
```

> **ดีเทลไทย:** ภาษาไทยไม่มีตัวพิมพ์ใหญ่ — eyebrow/label จึงแยกด้วย **น้ำหนัก+สี+ขนาด** ไม่ใช่ `text-transform: uppercase` หรือ letter-spacing แบบฝรั่ง

## 6. เลย์เอาต์ (Layout)

**mobile-first** (อาหารเดลิเวอรี = มือถือเกือบ 100%) จังหวะแนวตั้งเหมือน "เดินไปตามซอยตลาด" — แผงร้านโผล่มาทีละแผง

- คอนเทนต์เป็นการ์ด "แผงร้าน" (`--ink-card`) ลอยบนพื้นค่ำ มีระยะห่างหายใจ
- spacing scale ฐาน 4: `4 8 12 16 24 32 48 64`
- radius: การ์ด/รูปอาหาร **16px** (อาหาร = นุ่ม น่ากิน) · ป้าย/ชิปสถานะทรงเหลี่ยมคม (= ป้ายร้าน) — ผสมอย่างจงใจ ไม่ใช่ radius เดียวทั้งหน้า
- **แถบตะกร้าติดล่าง (sticky)** เรืองแสงพริก — เห็นยอดและปุ่มสั่งตลอด

---

> 🔗 **ตัวอย่างจริง:** [dual-rail-tracker.html](./dual-rail-tracker.html) — เปิดในเบราว์เซอร์ได้เลย มีปุ่มสลับสถานะ (กำลังทำ → กำลังส่ง → ถึงแล้ว/OTP → ร้านปฏิเสธ) หรือ deep-link ด้วย `#arrived`, `#rejected` · responsive + เคารพ reduced-motion

## 7. Signature: "รางคู่" — ตัวติดตามออเดอร์ (Dual-Rail Tracker)

สิ่งที่คนจะจำเว็ปนี้ได้ ไม่ใช่หน้าแรก แต่คือ **หน้าติดตามออเดอร์** — และเราทำให้มันเป็นภาพของ state machine จริงของเรา

จาก [order-lifecycle.md](../order-lifecycle.md) ออเดอร์มี **2 รางวิ่งคู่กัน** (ราง ร้าน × ราง ไรเดอร์) แล้วมาบรรจบที่ "ไรเดอร์รับอาหาร" — เราวาดมันเป็น **รางไฟนีออน 2 เส้นวิ่งขนาน** แล้วหลอมรวมเป็นเส้นเดียว นี่ไม่ใช่ของประดับ แต่ encode ความจริงของระบบ (Structure is information)

```
   ราง ร้าน 🔥                      ราง ไรเดอร์ 🛵
   ●━━━━━━━●━━━━━━━◍               ◍┄┄┄┄┄┄┄○┄┄┄┄┄┄┄○
   รับแล้ว  กำลังทำ  เสร็จ           รอคว้า   คว้าแล้ว  ถึงร้าน
   (เรืองมะม่วง = ทำอยู่)            (เรืองมะม่วง = กำลังมา)
            ╲                    ╱
             ╲                  ╱
              ◆ ไรเดอร์รับอาหาร ◆   ← จุดบรรจบ เรืองพริกตอน active
                     ┃
                     ●  กำลังส่ง → ถึงหน้าบ้าน
                     ┃
                  [ รหัสรับของ 4821 ]  ← โมเมนต์ OTP
```

- เส้นที่ "ทำอยู่/กำลังมา" เรืองมะม่วง, จุดที่ทำแล้วทึบ, จุดอนาคตเป็นจุดประจาง
- จุดบรรจบเรืองพริกตอนกำลังเกิด — เป็น climax ของหน้า
- รองรับทุก terminal: ปฏิเสธ/ยกเลิก/timeout/ส่งไม่ได้ → รางหยุดแล้วขึ้นการ์ดอธิบาย (ดูข้อ 9)

## 9. ภาษา & สถานะ (UX writing) — ผูกกับโดเมนจริง

เสียงแบรนด์: **เป็นกันเอง ตรงไปตรงมา ไม่ขอโทษพร่ำเพรื่อ** ปุ่มบอกสิ่งที่จะเกิด และคงชื่อเดิมทั้ง flow

| สถานะจริง (จาก lifecycle) | สิ่งที่ลูกค้าเห็น |
|---|---|
| ปุ่มสั่ง → ผลลัพธ์ | ปุ่ม "สั่งเลย" → toast "สั่งสำเร็จ" (ชื่อเดียวกันทั้งเส้น) |
| หน้าต่างยกเลิกฟรี | ชิป mono นับถอยหลัง "ยกเลิกฟรีได้อีก 4:32" |
| ร้านปฏิเสธ (ของหมด) | "ร้านของหมดพอดี — คืนเงินเต็มให้แล้ว ลองร้านใกล้ๆ ไหม" + ปุ่มร้านใกล้เคียง |
| ไม่มีไรเดอร์ (รอ) | "กำลังหาไรเดอร์ให้... ช่วงนี้คนสั่งเยอะ" (ไม่โกหกว่าเร็ว) |
| Delivery Timeout | "หาไรเดอร์ไม่ได้จริงๆ คืนเงินเต็มให้แล้ว ขอโทษด้วยนะ" |
| ส่งไม่ได้ (ติดต่อไม่ได้) | ก่อนถึงเกณฑ์: "ไรเดอร์รออยู่หน้าบ้าน รับสายด่วน!" |
| empty state ตะกร้า | "ตะกร้ายังว่าง — ตลาดเปิดรออยู่" + ปุ่มดูร้านใกล้ฉัน |

> error ไม่ขอโทษฟูมฟาย แต่บอก **เกิดอะไร + ทำต่อยังไง** เสมอ (เช่น ปฏิเสธ = บอกว่าคืนเงินแล้ว + เสนอทางไป)

## 10. Motion (จงใจ ไม่รก)

- **โหลดหน้าแรก:** ป้ายไฟ "ติด" ทีละนิด (flicker-on สั้นๆ ครั้งเดียว) เหมือนเปิดไฟแผง
- **รางคู่ (signature):** เส้นที่ active มี glow ไหลเบาๆ, จุดบรรจบ pulse ตอนกำลังเกิด
- **เพิ่มลงตะกร้า:** ของ "ลอย" เข้าแถบตะกร้าล่าง + ยอดเด้ง
- ที่เหลือ **นิ่ง** — ขยับเฉพาะจุดที่มีความหมาย
- เคารพ `prefers-reduced-motion`: ปิด flicker/ไหล เหลือแค่เปลี่ยนสถานะทันที

## 11. พื้นมาตรฐาน (ไม่ต้องป่าวประกาศ)

- responsive ลงจอมือถือเล็กสุด, แตะง่าย (เป้าแตะ ≥44px)
- focus คีย์บอร์ดเห็นชัด (เส้นเรืองมะม่วงรอบ element)
- contrast ผ่านเกณฑ์บนพื้นมืด (ดูข้อ 4)
- รูปอาหารมี alt บอกเมนูจริง

## 12. คำถาม/สิ่งที่อยากให้คุณเคาะต่อ

1. **ชื่อแบรนด์/โลโก้** — ป้ายไฟ hero ควรเป็นคำว่าอะไร? ตอนนี้ผมใส่ placeholder "หิวเมื่อไหร่ ตลาดเปิดเมื่อนั้น" ไว้
2. **ฟอนต์** — โอเคกับ Kanit + Anuphan + Space Mono ไหม (ทั้งหมดมีใน Google Fonts ฟรี) หรืออยากลองตัวอื่น
3. **ขั้นต่อไป** — อยากให้ผมทำ **HTML/CSS ตัวอย่างหน้า hero หรือหน้าติดตามออเดอร์ (รางคู่)** ให้เห็นของจริงไหม หรือเก็บเป็นเอกสารทิศทางไว้เท่านี้ก่อน
