# Product

## Register

product

## Users

แพลตฟอร์ม marketplace ส่งอาหารที่รับใช้ **4 ฝ่ายพร้อมกัน** — แต่ละฝ่ายอยู่คนละบริบทและมีงานหลักคนละอย่าง:

- **ลูกค้า** — เปิดแอปตอนหิว อยากได้ของเร็ว: ค้นร้าน → ปรับเมนู → สั่ง → ติดตามแบบ real-time → (ถ้ามีปัญหา) ร้องเรียนหลังส่ง
- **ร้าน (Merchant)** — หน้าจอทำงานระหว่างครัวยุ่ง: รับ/ปฏิเสธออเดอร์ที่เข้ามา, จัดการเมนู, เจรจาอัตราค่าคอมกับแอดมิน
- **ไรเดอร์ (Rider)** — ใช้บนถนน มือเดียว: คว้างานที่ว่าง (ใครกดก่อนได้ก่อน) → ถึงร้าน → รับอาหาร → ส่ง → ยืนยัน OTP
- **แอดมิน** — คุมภาพรวมระบบ: dashboard หลายออเดอร์, settlement/wallet, force-cancel, กำกับดูแล (ระงับ/ลดอันดับ), ตัดสินข้อร้องเรียน

## Product Purpose

เชื่อมทั้งสี่ฝ่ายในตลาดเดียว โดยให้ **ตรรกะโดเมนบริสุทธิ์** (state machine รางคู่ ร้าน×ไรเดอร์, settlement, wallet, dispute, moderation) เป็นแกนกลางที่ทั้ง web และ backend ใช้ร่วมกัน. ความสำเร็จวัดจาก: ออเดอร์เดินครบวงจรจริงผ่าน Postgres, แต่ละฝ่ายเห็นเฉพาะสิ่งที่ตัวเองกดได้/ควรรู้, และตัวเลขการเงินทุกบรรทัดตรวจย้อนได้จริง (ไม่มีตัวเลขปลอม).

## Brand Personality

**อบอุ่น มีชีวิตชีวา แบบตลาดกลางคืน — แต่เร็ว คล่อง ไม่ขวางทาง.**

ธีม "ตลาดกลางคืน": พื้นดำอมม่วงอุ่น, แสงป้ายไฟมะม่วง/พริก/ใบเตยเป็น accent, ให้ความรู้สึกสตรีทฟู้ดไทยยามค่ำ — ตรงข้ามกับแอปส่งอาหารสว่างจ้าที่ดูเหมือนกันหมด. บุคลิกนี้คือ *เปลือกที่จำได้*, แต่เมื่อถึงเวลาทำงานจริง (คอนโซลร้าน/ไรเดอร์/แอดมิน) ต้องหลบทางให้งานลื่น: feedback ทันมือแต่ไม่รก, motion มีวินัย.

3 คำ: **อบอุ่น · มีชีวิต · ไม่ขวางทาง**

## Anti-references

- **แอปส่งอาหารทั่วไป (สว่าง ขาว-ส้ม)** — look แบบ Grab/LineMan/foodpanda: พื้นขาวจ้า, ส้มสด, การ์ดเหมือนกันเรียงกันจนจำไม่ได้ว่าเป็นแอปไหน. นี่คือสิ่งที่ตั้งใจ *ไม่* เป็น.
- **SaaS dashboard เทมเพลต** — hero-metric การ์ดใหญ่, eyebrow ตัวเล็กทุก section, gradient text, sidebar generic.
- **หรูเกินจริง / คอร์ปอเรตแข็ง** — ดูแพงจัด เย็นชา เป็นทางการจนไม่เหมือนสตรีทฟู้ด.

## Design Principles

1. **รับใช้งาน ไม่ขวางทาง** — แต่ละหน้ารับใช้งานหลักของฝ่ายนั้น. แต่ละ actor เห็นเฉพาะ transition ที่ตัวเองกดได้ (merchantView/riderView ตัดสินให้) — ไม่ยัดปุ่มที่กดไม่ได้.
2. **ความจริงเดียว ไม่โกหกผู้ใช้** — ตัวเลขทุกอย่างมาจากสถานะจริง (stat hero ใช้ร้าน serviceable จริง, settlement โชว์ยอดสุทธิที่ตรวจย้อนได้). ไม่มีตัวเลข/สถิติ decorative ที่ไม่จริง.
3. **ตลาดกลางคืน ไม่ใช่แอปส่งอาหารทั่วไป** — ความอบอุ่น/มีชีวิตมาจากธีม dark warm + accent ป้ายไฟ + typography, ไม่ใช่การไล่ตาม convention สว่าง-ส้มของคู่แข่ง.
4. **optimistic แต่ซื่อสัตย์** — กดแล้ว UI ตอบทันที (local optimistic) แต่ถ้า mirror ไป backend ล้มเหลว → refetch ดึงความจริงกลับ + แจ้งผู้ใช้ตรงๆ (notice bar) ว่าอะไรพลาด. ไม่เงียบ ไม่แกล้งว่าสำเร็จ.
5. **วินัยของ signature** — glow/motion สงวนไว้ให้ "action หลักหนึ่งเดียวต่อหน้า" (ปุ่มร้อน/cartbar/สถานะ live). รอบๆ เงียบ — restraint คือสิ่งที่ทำให้ signature เด่น.

## Accessibility & Inclusion

- **ภาษาไทยเป็นหลัก** — typography เลือกจากข้อเท็จจริงว่าไทยไม่มีตัวพิมพ์ใหญ่ (แยกลำดับชั้นด้วยน้ำหนัก+สี ไม่ใช่ uppercase eyebrow).
- **contrast** — ตัวอักษรบนพื้นค่ำต้องผ่าน WCAG AA (body ≥4.5:1); ระวัง `--paper-dim` บนพื้นการ์ดเข้ม.
- **prefers-reduced-motion** — เคารพเต็มรูปแบบอยู่แล้ว (global rule ปิด animation/transition); ทุก motion ใหม่ต้องมีทางเลือก reduced.
- **focus ที่มองเห็น** — `:focus-visible` (outline มะม่วง) มีครบ; keyboard navigation ต้องใช้งานได้ทุกคอนโซล.
- **โหมดมือเดียว/กลางถนน (ไรเดอร์)** — ปุ่ม action ขนาดพอกด, ไม่พึ่ง hover.
